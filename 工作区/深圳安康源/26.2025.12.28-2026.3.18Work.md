```
_Generic 是 C 语言自 C 11 标准引入的一种编译时特性。它允许根据表达式的类型在多个表达式或语句之间进行选择。从本质上讲，它类似于一个基于类型的 switch 语句，但与传统 switch 不同的是，_Generic 是在编译时进行求值的，这使得它可以用于各种复杂的类型相关的场景。
```

KE 4002 隧道式血压计，充泄气模式。根据新板子新臂带结构的充泄气实测情况进行了调整。
原为：

```
①充气阶段 0 mmgh->230 mmgh：
    小于 15 s 报 E-3；大于 23 s 报 e-4
②漏气阶段 200 mmgh->40 mmgh：
    小于 47 s 报 E-5；大于 57 s 报 E-6
③快速泄气阶段 40 mmgh->15 mmgh：
    大于 3 s 泄气未到 15 mmgh 则报 E-7；大于 7 s 泄气到 0 mmgh 则报 E-8

现为：
①充气阶段 0 mmgh->230 mmgh：
    小于 10 s 报 E-3；大于 17 s 报 e-4
②漏气阶段 200 mmgh->40 mmgh：
    小于 35 s 报 E-5；大于 45 s 报 E-6
③快速泄气阶段 40 mmgh->15 mmgh：
    大于 4 s 泄气未到 15 mmgh 则报 E-7；大于 8 s 泄气到 0 mmgh 则报 E-8
```

##### 变量存储声明

```
在 C 代码中，使用 #define定义的是常量地址 ，不是变量存储。变量存储取决于如何声明变量。在 8051 C 中，变量可以放在不同内存区域：
Data: 内部 RAM 直接寻址
Idata: 内部 RAM 间接寻址
Xdata: 外部 RAM
Code: 程序内存
所以，这些宏可能用于指定变量的绝对地址。用户需要确保变量声明在正确的内存区域。

```

```
// IRAM变量（地址0x1E8–0x1FF）：使用idata或绝对地址定位
idata unsigned char ackData _at_ 0x1E8;  // 使用_at_关键字定位到绝对地址

// XRAM变量（地址0x100）：使用xdata

// IROM常量（地址0xEC00等）：使用code
code const char urlCheck[] _at_ 0xEC00;
```

##### 保身欣 S 603，V 106 中文语音。
在设置默认中，调用 SoundNoWait 指令后 while (！BUSY)语音播放过程中有按键判断、闪烁显示，而 KEY_MEMORYUP 键没有做退出判定，在按下按键后 BUSY 引脚的电平状态没有立马切换，导致多次发送 0 xFE 停止语音播报指令，且多次发送时波形是乱的 (发了十几次 0 xFE、中间没有延时)，误判为语音音量调节指令。
所以会出现打断语音后，下一个设置的语音播报音量会变。而变大后又变回去大概率是因为设置到了单位前是设置语音，会进行一次保存更新音量。

解决/排查思路: 尽可能模拟该情况下的语音播报。写一个 while (1)，在循环中发送“设置时间制式”的语音，且在播报后连续发送 100 次 0 xFE 停止播报的指令，中间不要有延时 delay。因为出问题的情况下就没有 delay。如果加了 delay，发送出去的波形是正常的，会复现不了这种音量突然变大的现象。



第三个 IO 口串联 1 个 1 K 电阻
总体超时检测复位，连续一直发，25 ms 视为超时。
指令间隔时间需要 25 ms
